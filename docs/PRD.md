# 课堂视频智能分析工具 — 产品需求文档（PRD）

## 1. 项目信息

- **Language**: 中文
- **Programming Language**: Python（CLI工具）+ Vite + React + MUI + Tailwind CSS（V2 Web界面）
- **Project Name**: `classroom_video_analyzer`
- **原始需求复述**: K12教育学习成长中心负责人日常需大量观看教师课堂视频进行质检、评分和素材提取，纯人工看视频每节课60分钟+，效率极低。需要一款基于语义驱动的智能分析工具，通过语音理解教学环节变化，结合画面分析，自动生成质检报告、评分卡和素材标注。

---

## 2. 产品定义

### 2.1 Product Goals

1. **效率10倍提升**：将单节课质检+评分时间从60分钟压缩至6分钟以内（人工审核报告即可），释放管理者精力
2. **标准化与一致性**：通过配置化的评分标准和质检清单，消除人工评分的主观偏差，确保不同人、不同时间的评估结果一致可复现
3. **优质素材高效沉淀**：自动识别并标注教学高光片段，让优秀教学案例的提取从"大海捞针"变为"精准定位"

### 2.2 User Stories

1. **As a** 学习成长中心负责人，**I want** 将课堂视频文件拖入工具后自动生成质检报告，**so that** 我不需要从头到尾看完整节课就能快速定位违规/遗漏环节
2. **As a** 教学质检员，**I want** 工具按我配置的评分标准对每节课多维度自动打分，**so that** 评分结果客观一致，不因我的状态或偏好而波动
3. **As a** 培训管理者，**I want** 工具自动标注课堂中的互动高光和教学亮点片段，**so that** 我能快速提取优质素材用于教师培训
4. **As a** 质检员，**I want** 工具以"先听后看"的语义驱动方式分析视频（先理解教学环节再截取关键帧），**so that** 分析结果贴合教学逻辑而非依赖画面猜测
5. **As a** 团队负责人，**I want** 评分标准和质检清单以配置文件形式管理，**so that** 一次校准后全团队长期复用，无需每次手动设定

## 3. 技术规范

### 3.1 Requirements Pool

#### P0 — Must Have（MVP核心）

| 编号 | 需求 | 说明 |
|------|------|------|
| P0-01 | 视频文件输入 | 支持本地视频文件（mp4/flv/mkv），单文件几百MB~1GB |
| P0-02 | FFmpeg音频提取 | 从视频文件中提取音频轨，输出为WAV/MP3供ASR处理 |
| P0-03 | 腾讯云ASR转文字 | 调用腾讯云语音识别API，输出带时间戳的文字转录 |
| P0-04 | 说话人分离 | 区分教师与学生语音，标注说话人切换节点 |
| P0-05 | 教学事件识别（LLM语义分析） | 基于带时间戳的转录文本，通过LLM识别6大类教学事件（环节切换/互动指令/学生应答/教师反馈/知识节点/节奏信号） |
| P0-06 | 事件驱动智能截帧 | 教学事件触发截帧，非固定间隔；事件发生前后截取关键帧 |
| P0-07 | 质检报告生成 | 基于语音+文本分析输出质检报告：环节完整性、互动频率、规范符合度 |
| P0-08 | 评分卡生成 | 按配置的评分标准和权重，输出多维度评分卡 |
| P0-09 | CLI命令行界面 | 单条命令完成全流程：`analyze <video_path> --config <config_path>` |
| P0-10 | 配置化评分标准 | 评分维度、权重、质检清单均为YAML/JSON配置文件 |

#### P1 — Should Have（V2增强）

| 编号 | 需求 | 说明 |
|------|------|------|
| P1-01 | 关键帧多模态分析 | 对截取的关键帧进行视觉分析：画面布局检测（主讲/共享屏幕/画廊/聚焦/白板）、教师状态、学生参与度 |
| P1-02 | 画面布局分类 | 识别5种画面模式：教师主讲大画面、共享屏幕/PPT、画廊模式、聚焦模式、互动白板 |
| P1-03 | 互动质量评估 | 结合语音语义+视觉信息，评估互动质量（提问深度、学生参与广度、教师反馈质量） |
| P1-04 | Web界面 | 提供浏览器操作界面，支持视频上传、配置管理、报告查看 |
| P1-05 | 时间轴可视化 | 在Web界面中展示教学事件时间轴，支持点击跳转到对应视频片段 |
| P1-06 | 批量处理 | 支持一次提交多个视频文件，队列化处理 |

#### P2 — Nice to Have（V3远景）

| 编号 | 需求 | 说明 |
|------|------|------|
| P2-01 | 高光片段自动剪辑 | 自动剪辑教学高光片段，输出可独立使用的短视频文件 |
| P2-02 | 学生参与度量化 | 基于语音活跃度+视觉信息，量化学生参与度指标 |
| P2-03 | 历史数据对比 | 同一教师不同课次的对比分析，识别教学改进趋势 |
| P2-04 | 多课程类型适配 | 支持不同学科、不同课型（新授课/复习课/实验课）的分析模板 |
| P2-05 | 报告导出 | 支持报告导出为PDF/Word格式 |

#### 配置文件结构（YAML）

```yaml
# scoring_config.yaml — 评分标准配置
scoring:
  dimensions:
    - name: 教学环节完整度
      weight: 0.25
      criteria: 开场→引入→讲授→练习→互动→总结，缺环节扣分
    - name: 互动质量
      weight: 0.25
      criteria: 提问频次、学生参与度、反馈质量
    - name: 知识讲解清晰度
      weight: 0.20
      criteria: 概念引入明确、例题典型、方法总结到位
    - name: 教学节奏
      weight: 0.15
      criteria: 节奏合理，无冗长停顿或赶进度
    - name: 规范符合度
      weight: 0.15
      criteria: 按教研规范执行

quality_checklist:
  - 有无开场白和课堂纪律提醒
  - 是否按时上下课
  - 互动环节是否有学生参与
  - 是否有知识总结
  - 有无不当言论

event_types:
  - 环节切换  # 开场/引入/讲授/练习/互动/总结
  - 互动指令  # 提问/举手/连麦/齐答/讨论
  - 学生应答  # 个人回答/集体回答
  - 教师反馈  # 肯定/纠正/追问/引导
  - 知识节点  # 概念引入/例题讲解/方法总结
  - 节奏信号  # 加速/放慢/重复/跳过
```

### 3.4 Open Questions

| 编号 | 问题 | 影响范围 | 建议跟进 |
|------|------|---------|---------|
| Q1 | 腾讯云ASR的说话人分离功能准确率是否满足需求？是否需要备选方案（如pyannote）？ | P0-04 | MVP阶段先用腾讯云，如准确率不足引入开源方案 |
| Q2 | LLM选型：语义分析用哪个模型？API调用还是本地部署？成本和延迟如何平衡？ | P0-05 | 建议MVP用GPT-4o/Claude API，后续可换本地模型 |
| Q3 | 教学事件识别的Prompt如何设计才能稳定输出结构化结果？需要多少轮校准？ | P0-05 | 需要准备5-10节样本视频进行Prompt调优 |
| Q4 | 单节视频完整分析耗时预期？ASR+LLM调用的成本预估？ | 整体 | 需实测：目标<6min，API成本<2元/节 |
| Q5 | 评分标准的初始配置由谁提供？是否需要内置默认模板？ | P0-10 | 波哥提供初始标准，工具内置3-4套通用模板 |
| Q6 | 视频画面模式复杂多变（5种+），V2关键帧分析如何处理模式切换？ | P1-01/P1-02 | 建议先用规则+LLM判断，后续可训练专用分类器 |
| Q7 | 多人同时说话时ASR和说话人分离如何处理？ | P0-03/P0-04 | 齐答场景可能识别困难，需设计降级策略 |

|------|------|-----------|---------|
| **MVP** | 语音+文本分析管线 | 质检70%+ / 评分60% | CLI工具 + 质检报告 + 评分卡 |
| **V2** | +关键帧视觉分析 | 质检90%+ / 评分80% | Web界面 + 时间轴可视化 + 批量处理 |
| **V3** | +高光片段自动剪辑 | 三场景全覆盖 | 视频剪辑导出 + 学生参与度量化 |

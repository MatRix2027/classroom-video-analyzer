# 项目决策记录

本文件用于记录影响产品方向、技术路线、评价标准或交付节奏的重要决策。

## 2026-06-15 接管项目并确立 Web 评价工具方向

- 背景：项目已从 WorkBuddy 迁移到当前工作区，已有视频上传、ASR、LLM 评分、视觉关键帧、Web 展示等基础能力，但缺少明确的项目立项和推进机制。
- 选项：
  - 继续把它作为一次性脚本工具维护。
  - 将它产品化为教师授课能力评价 Web 工具。
- 决策：将项目定位为“课堂视频教师授课能力评价 Web 工具”，优先完成单节课上传、分析、评分、报告和人工复核闭环。
- 原因：用户明确需要一个可用于教师授课能力评价的 Web 工具，而不是单纯的视频分析脚本。
- 影响：后续迭代以 Web 工作流和评价报告可用性为中心；高级能力如批量分析、趋势对比、素材剪辑放到后续阶段。
- 后续复盘点：当 10 节样本课完成校准后，复盘评分维度、Prompt、视觉模型和人工评分的一致性。

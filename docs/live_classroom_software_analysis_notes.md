# 火花直播课堂软件对分析逻辑的启发

来源：`火花常用软件介绍.pdf`。该资料说明教师端直播课堂包含课件区、教师区、学生区、菜单区、集体互动区、个体互动区、工具区，以及上台、加星、音频管理、画笔、抢答、摇号、投票、工单、摄像头/高拍仪切换等常用功能。

## 可进入分析逻辑的内容

1. 上台与互动机会
   - 识别视频/拖拽上台、课件上台、全屏上台、一键上台。
   - 用于判断学生表达机会、互动分布、是否集中在少数学生。

2. 激励与反馈
   - 识别集体加星、个体加星、台上学生加星。
   - 加星只有在有明确理由或紧跟具体表现时，才作为有效反馈证据。

3. 音频管理
   - 识别公麦、闭麦、小黑屋、私麦、个体音量调节。
   - 用于区分秩序维护、单点指导和不当压制。频繁或长时间闭麦作为风险信号。

4. 互动工具
   - 识别抢答、摇号、投票、小白板、调座位、分组。
   - 用于判断互动形式是否多样，但必须结合学生回应与教师反馈判断是否有效。

5. 板书与课件工具
   - 识别画笔、文本框、图形、橡皮擦、清空、课件标记、高拍仪/摄像头切换。
   - 用于支持板书课件质量、重点难点呈现和视觉讲解判断。

6. 技术异常处理
   - 识别刷新课堂、工单、学生/教师侧设备网络问题、麦克风/摄像头问题。
   - 用于判断教师组织教学与突发处理能力。

## 已落地到系统的改动

- `config/default.yaml`：新增事件类型 `直播工具操作`、`技术异常`。
- `prompts/event_detection.md`：新增直播课堂工具识别规则。
- `prompts/visual_observation.md`：新增第 9 类观察维度“直播软件界面与工具使用”。
- `prompts/quality_assessment.md`：新增“直播课堂软件操作证据”评分纪律，明确工具使用只作为证据，不直接等同于高质量教学。

## 后续可继续产品化的方向

- 在报告页增加“直播工具使用证据”模块，展示上台、加星、抢答、投票、闭麦、工单等事件时间线。
- 在人工校对入口增加“直播工具误判/漏判”反馈类型。
- 针对完整课堂输出“互动机会分布表”：学生上台次数、被点名次数、获得反馈次数、加星次数。
- 对技术异常形成单独提示：异常持续时间、影响对象、教师处理方式、是否影响教学主线。

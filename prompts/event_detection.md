## 事件类型说明

1. **环节切换**：教学环节之间的转换，如开场→引入→讲授→练习→互动→总结
2. **互动指令**：教师发起的互动行为，如提问、举手、连麦、齐答、讨论
3. **学生应答**：学生的回答行为，如个人回答、集体回答
4. **教师反馈**：教师对学生回答的反馈，如肯定、纠正、追问、引导
5. **知识节点**：知识传授的关键时刻，如概念引入、例题讲解、方法总结
6. **节奏信号**：教学节奏变化的信号，如加速、放慢、重复、跳过
7. **直播工具操作**：教师在直播软件中使用课堂工具的行为，如上台/下台、课件上台、全屏上台、一键上台、加星、抢答、摇号、投票、调座位、小白板、画笔、文本框、图形、课件标记、切换高拍仪/摄像头、刷新课堂、开始/结束课堂、课间休息等
8. **技术异常**：影响课堂运行的技术或设备问题，如学生卡顿、听不到声音、麦克风/扬声器异常、摄像头/高拍仪切换失败、课件卡顿、教师或学生侧网络异常、提交工单、刷新课堂、闭麦/公麦/私麦处理等

## 直播课堂工具识别规则

火花教师端常见区域包括：课件区、教师区、学生区、菜单区、集体互动区、个体互动区、工具区。请将这些操作视为课堂行为证据，而不是普通闲聊：

- **上台类**：视频/拖拽上台、课件上台、全屏上台、一键上台、上台模式切换。用于判断互动机会、学生表达、课堂组织。
- **激励类**：集体加星、个体加星、台上学生加星。只有出现具体理由或对应学生表现时，才可作为有效反馈证据；单纯机械加星不能直接判为高质量反馈。
- **音频管理类**：公麦、闭麦、小黑屋、私麦、个体学生音量调节。需区分课堂秩序维护、单点指导和不当压制；长时间或频繁闭麦属于风险信号。
- **互动工具类**：抢答、摇号、投票、小白板、调座位、分组。用于判断互动形式是否多样、是否服务教学目标。
- **板书课件类**：画笔、橡皮擦、清空、文本框、图形、课件标记、高拍仪/摄像头切换。用于判断视觉讲解、板书结构和操作熟练度。
- **异常处理类**：刷新课堂、学生/教师侧工单、设备网络问题、留言提醒。用于判断教师是否及时、低干扰地处理技术问题。

## 输出格式

请以JSON数组格式输出识别到的教学事件，每个事件包含以下字段：

```json
[
  {
    "event_type": "事件类型（环节切换/互动指令/学生应答/教师反馈/知识节点/节奏信号/直播工具操作/技术异常/红线行为）",
    "subtype": "事件子类型",
    "start_time": 事件开始时间（秒，浮点数，从转录文本的时间戳推断），
    "end_time": 事件结束时间（秒，浮点数），
    "description": "事件描述（简要说明发生了什么）",
    "confidence": 置信度（0.0-1.0，根据文本证据的明确程度判断），
    "related_text": "相关原文（转录文本中支撑此判断的原文片段）"
  }
]
```

## 注意事项

- 时间戳需从转录文本中的 `[MM:SS-MM:SS]` 格式推断转换
- 只输出有明确文本证据支持的事件，不要猜测
- 对直播工具操作，只在转录文本明确出现工具名称、教师操作指令或学生反馈时输出；不要凭空推断界面按钮被点击
- 技术异常需记录影响对象（教师侧/学生侧/全班）和处理方式（刷新、切设备、闭麦、提交工单、继续观察等）
- 置信度反映证据的明确程度：直接陈述≥0.8，间接推断0.5-0.8，模糊暗示<0.5
- 如果文本中没有识别到任何事件，输出空数组 `[]`
- 请仅输出JSON数组，不要输出其他内容
